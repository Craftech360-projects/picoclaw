## Findings (2026-07-18 test, 97 turns, 12 min)

**Latency — improved ~40% vs baseline**
- Voice-to-voice p50 3.7s / p95 5.0s (baseline: ~5.6s / ~11s)
- Drivers: VAD endpoint 800→600ms, persistent Sarvam TTS websocket, STT final now 1.5s avg
- Consistent across en/hi/kn/ml/ta/pa (3.1–4.1s avg per language)

**Open issues**
1. gemini-2.5-flash ignores identity rule ("I am a large language model", turns 5/7/18) and reply-language rule (English question → Kannada/Malayalam reply, turns 1/68/72/79) despite both rules verified present in the prompt. Escalation: test gemini-2.5-pro, or move rules into persona + shrink prompt.
2. VAD 600ms fragments slow speech — sentences split into separate turns (e.g. turns 20–22, 84–85). Consider 700ms.
3. Prompt tokens ~1,750/turn (vs ~29 completion) — main LLM cost driver; candidate for trimming/caching.

**Fixed and verified in this run**
- STT auto-detect always on (6 languages detected)
- TTS voice follows detected language per utterance
- Persistent TTS connection (config once per connection, ~130ms saved per sentence)
- Expression tags no longer spoken aloud
- Per-turn usage captured: STT audio seconds, LLM tokens, TTS characters
