What happens if minReplicas stays at 1?

HPA can scale from one warm agent pod to many pods, but it reacts after load appears. That works for gradual traffic. For a sudden crowd, users can wait while cold pods start.

The hidden requirement: node capacity

HPA creates pods. Nodes run pods. Current test uses one m7i.xlarge and one Picoclaw LiveKit pod.

Goal Needed capacity
6 users at 50% 1 pod
Pods per m7i.xlarge 1 production pod
Required nodes 1 m7i.xlarge node

Without enough nodes, extra pods stay Pending and cannot accept LiveKit jobs.

Recommended meaning of minReplicas=1

minReplicas=1 and maxReplicas=1 are okay for this one-node test. For a 20-, 50-, or 100-user event, increase node capacity and maxReplicas before the event instead of waiting for traffic to force it.

Model used: MAX_SESSIONS=12, HPA session target=50%, one observation slot equals 6 sessions per pod. Autoscaling is capped at 1 replica for the current test.
